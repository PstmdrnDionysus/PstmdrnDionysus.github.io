Import

The following section details the overall import process for TIR — Corporate.

Formatting Requirements

The following formatting requirements must be met when formatting an Excel file for import.

File Name Formatting

• File names may not begin with a numeral.
• File names may not contain spaces.
• File names may not contain special characters (including hyphens).
• File names may not contain duplicate names between multiple Excel files.
• File names in the Remote File Manager must be unique.
• Files may be .xls or .xlsx.
• .xls files may contain a maximum of 78 columns.

Spreadsheet Formatting

• Spreadsheets may not contain frozen panes.
• Spreadsheets may not contain hidden columns, rows, or fields.
• Spreadsheets may not contain formulas.
• Spreadsheet cells should be formatted as “Text.”
• Dollar amounts may be formatted as “Number.”
• Only the first tab within an Excel file will be read and imported.
• Reorder tabs to allow relevant data to appear in the first tab of the file.

Column Title/Header Formatting

• Column titles/headers may not be blank/null.
• Column titles/headers may not contain spaces.
• Column titles/headers may not contain more than 50 characters.
• Column titles/headers may not begin with a numeral (e.g. “1Box”).
• Column titles/headers may not contain duplicate column titles/headers.
• Column titles/headers are case sensitive.
• Changing titles/headers requires updating the import map.

Import File

To import a file, perform the following steps:

1.   From the Navigation Menu, select Data.

2.   Click Start Import.
3.   Click Excel File Import or 1220 File Import and click Next.

4.   To upload the desired import file, you can either drag and drop the file or click Choose File.

5.   If you have already uploaded a file, select the file from the Uploaded Files section.

6.   Once you have uploaded or selected a file, click Next.
7.   Select the type of record you want to import: only recipients, only form data, only payer, or any combination of payer, recipient data, and form data. The following are examples of the different import types:
a. To import a payer only, check the Payer box.

b.   To import recipient data only, select the appropriate option from the Recipient Data drop-down list.

c.   To import form data only, select the appropriate option from the Form Data drop-down list.

d.   To import any combination of payer, recipient data, and/or form data, check the Payer box and/or select the appropriate option(s) from the Recipient Data and Form Data drop-down lists, when applicable.

8.   The Record Type(s) you’ve selected will appear in the associated left-hand column.

9.   Click Import.
10.   Select a map that corresponds with the import file you’ve selected. You must select a map for each Record Type you’re importing.

11.   To add a new map for a given Record Type, click the icon associated with that Record Type.

12.   From the Add Map sliding panel, map the column headers from the Excel files you’ve uploaded to the fields within TIR - Corporate.
a. Enter a Map Name to save the map, and enter a Map Description if necessary.

b.   Map all Target fields listed in the Required section to the matching Source fields to save the map.

Create/Delete Rule

A rule, a term for an established relationship between two entities, is created when a Source field is mapped to a Target field. For example, when you select the Source TIN to map to the Target TIN, the TIN to TIN association becomes a rule.

To create/delete a rule, perform the following steps:

1.   Select a field name from the Source and Target sections.

2.   Another method is to click Add Rule; a new row will appear in the Rules section and the Source and Target field names that you want to map can be selected.

3.   To delete a rule, hover over the rule you want to delete and click the icon, followed by Delete.

4.   If the selected field names match, click Auto Map to compare the Source field names with the Target field names. This will automatically create a rule.

5.   You can apply the Function feature to a rule to transform the source data. To add a function, you must first create a rule. Once you’ve created a rule, the Function + field will appear.

 

Note: You can drag and drop a rule to reorder the list..

6.   Click the Function + field to view the Add Function sliding panel.

7.   From the Function Type drop-down menu, you can select Convert, Date, or Pad. Enter the necessary information and click Apply. Depending on the function you’ve selected, different field options will be provided.
a. Convert: This function allows you to convert data from the source file to a specific value for TIR - Corporate to accept.

b.   Date: From here, you can enter the format you want the date to be displayed in within the source file.

c.   Pad: Use this function to add characters to the left of a string of characters that is less than the length of what you are entering. The most common use case for Pad is adding a ‘0’ to the beginning of a TIN when Excel drops the leading zero.

8.   Once you’ve applied the appropriate function, that function will appear in the Rules section of the Add Map sliding panel.

9.   Click Save to save the map. The Import sliding panel will appear, displaying the maps you’ve just created.

10.   To edit a map, hover over the appropriate map and click the icon, followed by Edit.

11.   To select a map, click on the row of the map you want to add. This will display the map you’ve selected under the Data Map(s) column on the left side of the page.

12.   To search for a specific map, use the search bar associated with that map.

13.   Once you’ve selected all added maps and you see that they appear under the Data Maps column, click Next. The Map Preview information for the selected maps will be displayed.
14.   The Source field mappings, up to 10 rows, will be displayed in the Source Column, and the Target field mappings, up to 10 rows, will be displayed in the Target Column. You can view the Source Values below the Source Column and Target Column.

 

Note: You can drag and drop a rule to reorder the list.

15.   Prior to clicking Import, you can click Modify at any point to make any adjustments.

 

 

Import with Mappings

You can import Excel files in .xls or .xlsx format with mappings. IRS 1220 files are pre-mapped and cannot be imported this way.

1.   Click Import. If you’d like to cancel the import as it is loading, or at any time throughout the import process, click Cancel Import on the top right-hand corner of the page.

2.   Once an import is complete, the results will be displayed in the following four cards: New, Updated, Errors, and Failed. You can click on any card view a sliding panel that will display all records associated with the category you’ve selected.

a. New: The number of records successfully imported without errors.
b.   Updated: The number of records that already exist in the application, or the number of updates made to a record that is listed more than once in an import file.
c.   Errors: The number of records that have been successfully imported, but contain validation errors that may cause a transmittal rejection or mailing error.
d.   Failed: The number of records that were unsuccessfully imported due to errors that must be resolved in the import file and reimported.
